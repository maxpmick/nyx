# t50

## Overview
t50 is a high-performance, multi-protocol packet injector for *nix systems. It is designed for network infrastructure testing and can generate massive amounts of traffic across various protocols. It supports CIDR notation and advanced header manipulation for TCP, UDP, ICMP, IGMPv2, IGMPv3, EGP, DCCP, RSVP, RIPv1, RIPv2, GRE, ESP, AH, EIGRP, and OSPF. Category: Vulnerability Analysis / Stress Testing.

## Installation (if not already installed)
Assume t50 is already installed. If not:
```bash
sudo apt install t50
```

## Common Workflows

### Standard TCP SYN Flood
```bash
sudo t50 192.168.1.1 --flood --syn --dport 80
```

### Multi-protocol Sequential Stress Test
```bash
sudo t50 192.168.1.0/24 --protocol T50 --flood
```
Sends all supported protocols sequentially against the target range.

### GRE Encapsulated Traffic Simulation
```bash
sudo t50 10.0.0.1 --encapsulated --gre-saddr 192.168.1.5 --gre-daddr 10.0.0.1 --flood
```

### ICMP Destination Unreachable Simulation
```bash
sudo t50 192.168.1.1 --protocol ICMP --icmp-type 3 --icmp-code 1 --threshold 500
```

## Notes
- **Root Required**: t50 requires root privileges to craft raw packets.
- **Performance**: The `--turbo` flag significantly increases packet output but may consume high CPU/bandwidth.
- **Randomization**: Many fields default to `RANDOM`. To set a field to zero explicitly, you must specify `0`.
- **Protocol T50**: Using `--protocol T50` cycles through all supported protocols.
- **Safety**: This tool is extremely powerful and can easily disrupt network services. Use only on authorized targets.
