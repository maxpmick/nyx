# vinetto

## Overview
Vinetto is a forensics tool designed to examine Thumbs.db files and Extensible Storage Engine (ESE) databases generated by Microsoft Windows. It allows investigators to extract thumbnail images and associated metadata (filenames, dates, paths), even if the original images have been deleted from the system. Category: Digital Forensics.

## Installation (if not already installed)
Assume vinetto is already installed. If the command is missing:

```bash
sudo apt install vinetto
```

## Common Workflows

### Basic file extraction
Extract thumbnails from a specific Thumbs.db file into a directory:
```bash
vinetto -o ./output_dir Thumbs.db
```

### Recursive directory processing
Search for and process all Thumbs.db files within a directory structure:
```bash
vinetto -m r -o ./forensics_report /mnt/windows_drive/Users/
```

### Generating an HTML report
Extract thumbnails and generate a visual HTML report for easier review:
```bash
vinetto -H -o ./report_folder Thumbs.db
```

### Using a Windows EDB file for metadata
Use a Windows.edb file to attempt to resolve original filenames for the thumbnails:
```bash
vinetto -e Windows.edb -o ./output Thumbs.db
```

## Complete Command Reference

```
vinetto [-h] [-e EDBFILE] [-H] [-m [{f,d,r,a}]] [--md5] [--nomd5] [-o DIR] [-q] [-s] [-U] [-v] [--version] [infile]
```

### Positional Arguments

| Argument | Description |
|----------|-------------|
| `infile` | Depending on operating mode, either a location to a thumbnail file ("Thumbs.db" or similar) or a directory. |

### Options

| Flag | Description |
|------|-------------|
| `-h`, `-?`, `--help` | Show help message and exit. Use `-v` for more details. |
| `-e`, `--edb EDBFILE` | Examine EDBFILE (Extensible Storage Engine Database) for original thumbnail filenames. `-e` without an `INFILE` explores EDBFILE extracted data. Automatic mode will attempt to use ESEDB without `-e`. |
| `-H`, `--htmlrep` | Write HTML report to DIR (requires option `-o`). |
| `-m`, `--mode [{f,d,r,a}]` | Operating mode:<br>`f`: Single file processing (default)<br>`d`: Directory processing<br>`r`: Recursive directory processing from a starting directory<br>`a`: Automatic processing using well-known directories starting from a base directory. |
| `--md5` | Force MD5 hash calculation for an input file. Normally calculated if file is < 0.5 GiB. |
| `--nomd5` | Skip MD5 hash value calculation for an input file. Overrides `--md5`. |
| `-o`, `--outdir DIR` | Write thumbnails to DIR. Requires `INFILE`. |
| `-q`, `--quiet` | Quiet output: Errors only. (Note: `-v` overrides `-q`). |
| `-s`, `--symlinks` | Create symlink from the image realname to the numbered name in `DIR/.thumbs` (requires `-o`). Requires a Catalog or `-e` EDB file to produce results. |
| `-U`, `--utf8` | Use UTF-8 encodings. |
| `-v`, `--verbose` | Verbose output. Each use increments level: 0 (Standard), 1 (Verbose), 2 (Enhanced), 3 (Full). |
| `--version` | Show program's version number and exit. |

## Notes
- **Metadata Recovery**: Thumbs.db files often persist after the original images are deleted, making this tool critical for proving the prior existence of files.
- **Output Structure**: When using `-o`, vinetto typically creates a `.thumbs` subdirectory containing the actual image files.
- **ESE Databases**: Modern Windows versions (Vista and later) use `thumbcache_*.db` files and the `Windows.edb` index; vinetto's `-e` flag is designed to handle these newer formats.
